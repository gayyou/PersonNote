### 1.多线程与单线程语言的区别

说起多线程语言和单线程语言，有两门语言很相近，但是区别却比较大，它们就是**Java**和**JavaScript**。我觉得将这两门语言来比较的话，能够代表大多数单线程和多线程语言的比较。首先，以我个人的经历的话，我是接触过这两门语言，那在我脑海里，这两门语言的区别在哪？

- **JavaScript**：动态、弱类型语言。并且是单线程语言，也就是一个进程中只有一个用户线程，不可以有多个线程。用户（程序员）在开发的时候，无需考虑到多个线程之间的资源竞争（前端并不需要，但是写**Node**后端的话，还是要考虑多进程之间共享公共资源）。只需要考虑一个线程，（打个比方就是做一件事情，比如生产500件物品需要用到一个工具，因为只有一个人来干活，所以不需要考虑在某个时间段工具被两个人以上进行争夺的情况。），在同步代码很符合人的思维逻辑。但是在异步代码中却会比较绕。**JavaScript**是单线程语言，要实现并发的话只能通过异步来进行实现。而**JavaScript**对于异步操作何时回调的话，使用了事件循环机制。不过这些理解起来会比较容易。
- **Java**：静态、强类型语言。是个多线程语言。在一个进程中拥有多条线程，这好比一个工厂里面多个人同时生产500件物品，如果工具只有一件的话，需要考虑到谁能够使用这个工具。是一个并行的世界。个符合客观事实（我们的操作系统也是一个并行的世界），要通过各种机制实现资源的合理分配。

### 3.什么是原子操作？

在**cpu**计算中不可再分的操作。由于**cpu**进行计算的话，是以时间片轮转算法来进行各个进程之间的调度的。而**java**虚拟机中线程的操作最终也会映射到**cpu**的操作。虚拟机会令原子操作执行**要么不做，要么全做**的操作。并且在执行的过程中其他线程是无法进行任何（可以限定区域）操作的。
